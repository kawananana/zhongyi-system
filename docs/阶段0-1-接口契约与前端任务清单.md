# 本草萌智 · 阶段 0 & 1 接口契约与前端任务清单

> **文档版本**：1.0  
> **对应需求**：需求分析报告 2.0 · §4 功能需求 · §6 数据字典（P1/P2）· §7 数据库设计 · §8 接口  
> **阶段定义**  
> - **阶段 0**：底座 · 用户与健康管理（P1）  
> - **阶段 1**：学 · 科普与图鉴引擎（P2）  
> **技术栈**：Vue 3 + Vite（C 端/B 端）· Spring Boot + MySQL + Redis + JWT  

---

## 全局约定

| 项 | 约定 |
|----|------|
| Base URL | `/api/v1` |
| 鉴权 | 需登录接口：`Authorization: Bearer {accessToken}` |
| 统一响应 | `{ code, message, data, timestamp }` |
| 分页结构 | `data: { list, total, page, pageSize }` |
| 常见错误码 | `401` 未登录/Token 失效；`403` 无权限；`422` 参数校验失败 |

## 一、阶段 0：底座 · 用户与健康管理（P1）

### 1.1 身份认证鉴权（数据流 1.1 · JWT 下发）

| 接口名称 | 方法 | URL | 鉴权 | 核心入参 | 返回 `data` 核心字段 |
|----------|------|-----|------|----------|----------------------|
| 用户注册 | POST | `/auth/register` | 否 | `phone`, `password`, `nickname?`, `smsCode?` | `userId`, `nickname` |
| 用户登录 | POST | `/auth/login` | 否 | `phone`, `password` | **`accessToken`**, **`refreshToken`**, `expiresIn`, `tokenType`(`Bearer`), `userBrief`: `{ userId, nickname, avatar }` |
| 第三方授权登录 | POST | `/auth/oauth/{provider}` | 否 | `provider`(`wechat`/`alipay`), `authCode` | 同登录 |
| 刷新 Token | POST | `/auth/refresh` | Refresh | `refreshToken` | 新 `accessToken`, `expiresIn` |
| 退出登录 | POST | `/auth/logout` | 是 | — | `success` |
| 发送找回密码验证码 | POST | `/auth/password/sms` | 否 | `phone` | `expireSeconds` |
| 重置密码 | POST | `/auth/password/reset` | 否 | `phone`, `smsCode`, `newPassword` | `success` |
| 获取当前用户资料 | GET | `/user/profile` | 是 | — | `userId`, `nickname`, `phone`(脱敏), `avatar`, `gender`, `birthday`, `status`, `createTime` |
| 更新用户资料 | PUT | `/user/profile` | 是 | `nickname?`, `avatar?`, `gender?`, `birthday?` | 更新后完整 profile |

#### Token 契约（联调重点）

| 项 | 约定 |
|----|------|
| 登录成功 | 必须返回 `accessToken` + `refreshToken` |
| 请求携带 | `Authorization: Bearer {accessToken}` |
| 过期策略 | `accessToken` 建议 2h；`refreshToken` 建议 7d |
| 401 处理 | 调用 `/auth/refresh` 静默刷新；失败跳转 `/login` |
| 密码 | 后端 BCrypt；传输走 HTTPS |
| 前端状态 | Pinia `useUserStore`：`token`, `userBrief`, `isLoggedIn` |

### 1.2 健康数据录入（数据流 1.2）

| 接口名称 | 方法 | URL | 鉴权 | 核心入参 | 返回 `data` 核心字段 |
|----------|------|-----|------|----------|----------------------|
| 获取健康档案 | GET | `/health/archive` | 是 | — | `archiveId`, `userId`, `sleepQuality`, `sleepHours?`, `examInfo`, `bmi`, `medicalHistory`, `allergyHistory`, `constitutionType`, `updateTime` |
| 保存/更新健康档案 | PUT | `/health/archive` | 是 | `sleepQuality`, `sleepHours?`, `examInfo`, `height`, `weight`, `medicalHistory`, `allergyHistory` | 档案 + 服务端计算 `bmi` |
| 触发体质分析 | POST | `/health/constitution/analyze` | 是 | `forceRefresh?` | `constitutionType`, `constitutionLabel`, `reportSummary`, `suggestions[]`, `generatedAt` |
| 获取体质分析报告 | GET | `/health/constitution/report` | 是 | — | 同上（缓存报告） |

**校验要点**：`bmi` 由身高(cm)、体重(kg) 服务端计算；`examInfo` 为结构化 JSON；非法数值返回 422。

### 2.2 多维复合过滤（数据流 2.2 · 重点）

| 接口名称 | 方法 | URL | 鉴权 | 核心入参 | 返回 `data` 核心字段 |
|----------|------|-----|------|----------|----------------------|
| 图鉴复合检索 | GET | `/atlas/herbs/search` | 否 | **FilterQuery**（见下表） | 分页 `list[]`, `total` |
| 图鉴默认列表 | GET | `/atlas/herbs` | 否 | `page`, `pageSize`, `sort?` | 同上 |
| 药材详情 | GET | `/atlas/herbs/{herbId}` | 否 | — | 完整图鉴 + `images[]` + `viewCount`, `collectCount`, `isCollected?` |
| 检索筛选项字典 | GET | `/atlas/herbs/filter-options` | 否 | — | 药性/药味/归经/省份等枚举 |

#### FilterQuery 条件结构（对齐 §6.2.2「检索标签」）

**请求**：`GET /atlas/herbs/search`

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `keyword` | string | 否 | 模糊：药材名、别名 |
| `provinceCodes` | string | 否 | 产地 ID，逗号分隔，如 `11,44,51` |
| `natures` | string | 否 | 药性多选：`寒,凉,平,温,热` |
| `tastes` | string | 否 | 药味多选：`酸,苦,甘,辛,咸` |
| `meridians` | string | 否 | 归经多选，如 `肝,脾` |
| `daoDiOnly` | boolean | 否 | 仅道地药材 |
| `page` | int | 是 | 默认 1 |
| `pageSize` | int | 是 | 默认 20，最大 50 |
| `sort` | string | 否 | `default` / `view` / `collect` |

**组合逻辑**：维度间 **AND**；同维度多值 **OR**（例：`natures=寒,凉` → 寒或凉）。

**列表项字段**：`herbId`, `herbName`, `alias`, `coverImage`, `originProvince`, `daoDiRegion`, `nature`, `taste`, `meridianSummary`, `efficacyBrief`, `viewCount`, `collectCount`

### 3.4 联调检查点

| 场景 | 检查项 |
|------|--------|
| 登录 | 含 `accessToken` / `refreshToken`；后续请求 Bearer |
| Token 过期 | 401 → refresh → 重放；失败跳登录 |
| 多维检索 | `provinceCodes` + `natures` + `keyword` AND/OR 正确 |
| 未登录收藏 | 引导登录；`batch status` 回显 |
| 地图点击 | `provinceCode` 与检索参数编码一致 |
| 3D 页 | 模型 CDN 可访问；点击拉穴位详情 |
